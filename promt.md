我设计了一款基于 STM32G030F6P6 (TSSOP-20封装) 的微型 24V 电池电压监控与智能报警板。系统运行频率为 64MHz (内部 HSI 配合 PLL)。
我已经使用 STM32CubeMX 生成了初始化工程。现在需要你帮我编写各功能模块的 C++ 语言代码（基于 HAL 库）。

# 硬件引脚分配与外设映射
- **调试心跳灯**: PA7 (GPIO Output)
- **按键**: PC14 (GPIO Input, 内部上拉，外部有 100nF 消抖电容，低电平有效)
- **ADC 电池采样**: PA0 (ADC1_IN0)。外部通过 100k 和 10k 电阻对 24V 电池分压（分压比 11:1）。
- **SPI Flash (W25Q64)**:
    - SPI1_SCK: PA1
    - SPI1_MOSI: PA2
    - SPI1_MISO: PA6
    - FLASH_CS: PA4 (GPIO Output)
- **音频功放控制**: PA3 (AMP_SHDN, GPIO Output, 高电平工作，低电平静音关断)
- **音频 PWM 输出**: PA8 (TIM1_CH1)。使用高频 PWM + 外部 RC 一阶低通滤波(1kΩ+10nF)模拟 DAC。
- **WS2812 RGB 灯**: PA5 (GPIO Output, 运行频率要求极高)
- **UART 通信**: PB6 (USART1_TX), PB7 (USART1_RX)。用于打印 Log 和后续 YMODEM 升级音频。
- **I2C 通信**: PA11 (I2C2_SCL), PA12 (I2C2_SDA)。(暂作预留)

- **MWL 中间件层 (Middleware Layer)**

            一些高层次（脱离硬件）可复用组件。
            e.g. 算法实现，数据结构实现...

    - **APL 应用层 (Application Layer)**

            实现程序执行逻辑。  
            e.g. 主函数实现（初始化、循环），回调函数实现。

    - **FML 功能模块层 (Function Module Layer)**

            通过调用 HDL、HAL 层封装的外设接口，组合成对应功能模块，实现相应功能。   
            e.g. 底盘功能模块实现，云台功能模块实现。

    - **HDL 硬件驱动层 (Hardware Driver Layer)**

            通过调用 HAL 层封装片上外设接口，实现对片外外设驱动。  
            e.g. 板级支持包实现，电机驱动实现...
        - Motor_DJI (大疆电机驱动)
        - Motor_DM (达妙电机驱动)

    - **HAL 硬件抽象层 (Hardware Abstract Layer)**

            实现对于片上外设再次封装。  
            e.g. CAN外设封装，USART外设封装...

3. SPI Flash 读写功能. 实现对 W25Q64 SPI Flash 的基本读写功能。同时你需要设计一下该flash中的数据存储结构. 我要求flash中最开始的部分存储音频数据的起始位置和结束位置(应当至少支持存储30段音频). 此后的部分存储的是音频数据. 为了保证音频顺利播放, 你可能需要通过dma配合双缓冲区等手段保证数据读取速度.
4. 音频播放功能. 通过 PWM 输出模拟 DAC 的方式实现音频播放功能. 我希望16kHz 采样率 + 8-bit 的音频。你需要实现一个简单的音频数据格式，并编写相应的解码函数将存储在 SPI Flash 中的音频数据解码为 PWM 占空比值进行输出。请注意我的扬声器额定功率1W, 最大功率1.5W, 请提供一定的保护.
5. WS2812 RGB 灯控制功能. 实现对 WS2812 RGB 灯的控制功能。你需要编写一个函数，接受 RGB 颜色值作为输入，并通过 GPIO 输出相应的信号来控制 WS2812 灯的颜色显示。需要避免使用到ws2812的最大功率, 因为那会对电路产生很大压力.
6. 用户按键功能: 按下按键后60s内不进行任何音频播放.
7. 电池低压报警功能: 如果iic能够正常读取电池架数据, 那么以获得的电池电量百分比为依据, 低于35%提供报警信号. 如果iic没有接入, 则以adc获得的电压为依据, 低于21.5V提供报警信号. 报警信号包括播放音频(或者音频组合, 当前阶段请默认循环播放第一段音频, 如果第一段音频地址无效, 请播放440Hz的音符), 以及ws2812红色灯光闪烁(正常时使用绿色常亮).
8. UART音频传输功能. 可以参考TCP的累计确认机制配合校验和, 通过一系列数据包(推荐单个包长度128字节)来完成音频的传输. 你需要在注释中声明上位机协议, 但目前不需要实现上位机功能.
